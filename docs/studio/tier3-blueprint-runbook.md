# Tier 3 · New blueprint runbook

Engagement: **0→1 blueprint** (€85k baseline, 14–24 weeks). Console preset: `tier3-blueprint`.

## Extra steps before provision

1. **Blueprint discovery locked** — schema, flows, scope doc signed with client
2. **Architecture signed** — technical design + milestone plan agreed

Only then stamp tenant / scaffold (template may not exist until mid-project).

## Template spec

When the first product template is defined, complete **template spec locked** (same as Tier 2).

## Dual sign-off

Before stamp: **Blueprint discovery** and **Architecture & milestones** each need client sign-off (portal Pulse) and studio sign-off (Console Delivery). Use the **Tier 3 artifact drafts** on Delivery to generate scope + architecture markdown, then lock gates when both parties agree.

**Template spec locked** (mid-project) uses the same dual sign-off pattern as Tier 2.

## Automation

- Longest Factory runbook — all phases on deal **Delivery** + T3 draft copy panel
- Desk prioritises discovery → architecture → tenant → template spec → build → handover
- 48h Slack/email if any runbook step blocks (Desk load, deal Delivery, or `pnpm studio:runbook-alerts`)
