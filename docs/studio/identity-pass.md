# Identity pass

**Identity** = look and voice within the shipped template’s layout patterns. Not new flows or entities.

Complete this pass after **tenant stamped** and before heavy feature build.

## Checklist (mirrored in Console)

1. **Client app brand config** — `app.config.ts` (or equivalent): name, tagline, primary colour, support email, deep links.
2. **Template marketing copy** — Console → Catalog → **Template copy** (taglines, hero, starts-at if shown).
3. **Admin surfaces branded** — Tenant name, accent, login copy in Admin.
4. **Client assets** — Logo, palette, fonts received (or studio flat-fee line delivered).

## Out of scope without change order

- New screens or navigation models
- Custom auth providers
- White-label layout engine
- Net-new illustration systems beyond agreed count

## Console

Deal → **Delivery** → Factory runbook → **Identity** phase → sub-checklist toggles.

When all sub-items are checked (or **Mark all done**), the Identity step completes and Desk advances to Configuration.
